# SaaS Agentic Interior ERP — UI System Guide

## 1. UI Foundation

This project uses shadcn/ui with a custom preset.

```bash
bunx --bun shadcn@latest init --preset b5d2ZsMLI --template next --pointer
```

```txt
UI Library: shadcn/ui
Preset: b5d2ZsMLI
Template: Next.js
Package Manager: Bun
Local Runtime: Bun
Styling: Tailwind CSS + CSS variables
```

---

## 2. Source of Truth

The shadcn preset is the UI source of truth for:

- Typography
- Radius
- Spacing
- Color tokens
- Border treatment
- Shadows
- Form style
- Table style
- Interaction states
- Base components

Do not create a second design system.

---

## 3. Bun Rules

Use Bun for all package and CLI commands.

```bash
bun install
bun dev
bun run build
bun run lint
bunx --bun shadcn@latest add button card table form input select
```

Rules:

- Do not use npm, pnpm, or yarn unless explicitly requested.
- Do not generate `package-lock.json`, `pnpm-lock.yaml`, or `yarn.lock`.
- Keep `bun.lock` or `bun.lockb` as the source of truth.
- Use `bunx --bun` for shadcn CLI.

## 4. Visual Direction

The ERP should feel:

```txt
Clean
Quiet
Premium
Professional
Readable
Operational
Data-dense
Calm
```

The desired reference vibe:

```txt
Linear + Notion + Vercel dashboard, with Dekoria premium taste.
```

Avoid:

- Overly colorful admin templates
- Heavy gradients
- Random animations
- Dashboard clutter
- Marketing landing page styling inside ERP pages
- Hardcoded brand colors not connected to tokens

## 8. ERP UI Patterns

### Metric Card

Use for dashboard summary numbers.

Content:

- Label
- Value
- Small description
- Optional trend/status

### Status Badge

Use consistent semantic badges for:

- Project status
- Project health
- Design status
- Material urgency
- Lead status
- Content status

### Data Table

Use for operational lists.

Expected table features:

- Search
- Filter
- Status badge
- Row actions
- Empty state
- Loading skeleton
- Pagination later if needed

### Forms

Use:

- React Hook Form
- Zod validation
- shadcn Form components
- Clear error messages
- Minimal fields per screen

---

## 9. UI Generation Rules for AI Agents

When generating UI:

- Use shadcn components first.
- Follow the existing preset.
- Do not overwrite `components/ui` casually.
- Create custom components in `components/shared` or module folders.
- Keep screens clean and operational.
- Use semantic status badges.
- Avoid unnecessary animations in ERP screens.
- Do not hardcode random colors.
- Use theme tokens and CSS variables from the preset.
- Prefer composition over one-off custom UI.

---

## 10. UI Success Criteria

The MVP UI is successful if:

- Owner can understand company condition within 30 seconds.
- PM can submit daily update quickly.
- Designer can update DED/design status without confusion.
- Purchasing can flag urgent material issues.
- Sales can update leads simply.
- Marketing can find content-ready projects.
- AI summary feels like a serious operational briefing, not a generic chatbot answer.
